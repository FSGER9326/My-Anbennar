- `memorySearch.provider: "ollama"` — stale config entry, but lancedb is disabled so Ollama is never called
- `memorySearch.model: "nomic-embed-text"` — same, stale

### Why config.patch fails for memorySearch.provider
- `config.patch` does full-object replacement at the patched key path
- The `memorySearch` schema declares `provider` as writeable only via specific allowed values, but the actual live config has additional fields (`model`) that aren't in the schema
- Attempted patches: `{"enabled":false}` → schema error; `{"provider":"builtin","model":null}` → schema error
- **Safe approach:** leave stale `memorySearch` entry as-is (harmless), no full config rewrite
