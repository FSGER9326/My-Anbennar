### Plugin SIGKILL Crashes — Root Cause Found
- System logs show: `SIGKILL` at `browser-lease-manager/index.js:169:15`
- Also crashed on second attempt
- **Bug #1:** `browser-lease-manager` register is `async` — OpenClaw ignores returned promises, all CLI/http registrations never fire
- **Bug #2:** `browser-signal-registry` uses `registerCommand` with subcommands array — wrong API format; should use `registerCli` with `program.command()`

### Fixes Applied
1. **browser-signal-registry** — completely rewritten:
   - Changed `async register(api)` → `register(api)` (sync)
   - Changed `registerCommand({ subcommands: [...] })` → `registerCli(({ program }) => { program.command(...) })`
   - Removed `async` from all handlers (sync API)
   - CLI: `openclaw signal list`, `openclaw signal show <site>`, `openclaw signal check <url>`
   - HTTP: `GET /plugins/browser-signal-registry/signals`

2. **browser-lease-manager** — NOT YET FIXED:
   - File still has `async register(api)` — needs `async` removed
   - The file needs: `register(api)` (remove `async`)
   - All the CLI and HTTP registrations inside are correct; just the `async` wrapper needs removing
