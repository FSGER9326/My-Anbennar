# EU4 Local Reference Index

## Purpose
This file tracks the local saved EU4 wiki pages that can be used as syntax and system references during Verne overhaul work.

These pages are local HTML snapshots stored in:

- `C:\Users\User\Documents\GitHub\My-Anbennar\EU4 Wiki references`

They are useful for:

- engine syntax reminders
- trigger and effect reference
- system-specific modding rules
- implementation sanity checks before coding

They are **not** a replacement for reading the Anbennar repo first.

## Trust order
When working on the mod, use this order:

1. existing Anbennar repo pattern
2. local EU4 wiki snapshot pages
3. live wiki / web lookup if the local snapshot is unclear or incomplete
4. memory only if the above are insufficient

## Known strengths for Verne work
These local references should be especially helpful when we need:

- `Triggers`, `Effects`, `Scopes`, and `Variables` for helper-layer scripting
- `Mission modding` and `List of common mission triggers` for mission rewrites
- `Idea group modding` and `Policy modding` for doctrine systems
- `Government modding` for reforms and state-path logic
- `Great project modding` for monument work
- `Mercenaries modding` for order companies
- `On Actions`, `Event modding`, and `Decision modding` for dynasty/court systems
- `Modifier list` when we need valid modifier names instead of guessed ones

## Practical rule for sessions
If a coding task needs syntax or system rules and the repo pattern alone is not enough, say which of these local pages was consulted before coding.

That makes the grounding pass easier to verify later.
